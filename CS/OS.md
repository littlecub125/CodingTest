# 페이징이란?
페이징은 CPU가 사용하는 프로세스를 더 효율적으로 관리하기 위해 메모리를 관리하는 기법입니다. 프로세스를 일정한 크기의 페이지로 나누고, 이러한 페이지들을 RAM과 보조 기억 장치(예: 하드 디스크) 사이에 교환합니다. 
RAM이 가득 차거나 단편화 문제가 발생할 수 있으므로, 사용되지 않는 페이지는 보조 기억 장치로 이동시키고, 필요한 페이지를 RAM에 적재합니다.

페이징을 통해 프로세스의 논리적 주소를 물리적 주소로 변환하며, 이 변환 작업은 페이지 테이블을 통해 이루어집니다. 논리적 주소는 CPU가 참조하는 주소이며, 페이지 테이블은 이 주소를 실제 물리적 메모리 주소로 매핑합니다. 
이로써 RAM의 효율적인 사용과 메모리 단편화 문제를 해결할 수 있습니다.

### 주요개념
1. 논리적주소
   논리적 주소: 프로세스가 CPU에 의해 생성되는 주소입니다. 프로그램이 사용하는 주소 공간을 말합니다.
2. 물리적주소
   물리적 주소: 실제 메모리(RAM)에서의 주소입니다. 논리적 주소는 페이지 테이블을 통해 물리적 주소로 매핑됩니다.
3. 페이지
   프로세스의 주소 공간을 일정한 크기로 나눈 작은 블록입니다.
5. 프레임
   물리적 메모리를 페이지와 같은 크기로 나눈 블록입니다.
7. 페이지 테이블
   각 프로세스마다 존재하며, 논리적 주소를 물리적 주소로 변환하는 데 사용됩니다. </br>
   페이지 번호를 이용하여 해당 페이지가 물리적 메모리의 어느 프레임에 위치하는지를 나타냅니다.

### 동작원리
1. 페이지 교환
   프로세스가 실행될 때 필요한 페이지가 메모리에 없으면, 운영 체제는 그 페이지를 디스크에서 메모리로 가져옵니다. </br>
  메모리가 꽉 차 있으면, 운영 체제는 사용하지 않는 페이지를 디스크로 내보내고, 새로운 페이지를 적재합니다. 이를 페이지 교환이라고 합니다.
2. 주소 변환
   CPU가 프로세스를 실행하면서 논리적 주소를 참조하면, 페이지 테이블을 통해 해당 논리적 주소를 물리적 주소로 변환합니다. </br>
이 과정에서 페이지 번호와 오프셋(offset)이 사용됩니다. 페이지 번호는 페이지 테이블을 참조하는 데 사용되고, 오프셋은 페이지 내에서의 위치를 나타냅니다.
